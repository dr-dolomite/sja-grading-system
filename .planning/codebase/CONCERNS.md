# Codebase Concerns

**Analysis Date:** 2026-03-30

## Tech Debt

**Unbuilt Application (Scaffold Only):**
- Issue: The project is a raw `create-next-app` scaffold. No grading system logic exists. `app/page.tsx` is the default template page with Vercel/Next.js promotional links. No domain models, no data layer, no routes, no API endpoints, and no business logic have been implemented.
- Files: `app/page.tsx`, `app/layout.tsx`
- Impact: The application does nothing. Any "start here" work requires replacing the entire page content.
- Fix approach: Replace `app/page.tsx` entirely. Define the application's route structure and begin implementing grading system features.

**Unused Font Registrations:**
- Issue: Three fonts are loaded in `app/layout.tsx` — Geist Sans, Geist Mono, and Manrope. Only Manrope is wired as `--font-sans`. Geist Sans (`--font-geist-sans`) and Geist Mono (`--font-geist-mono`) inject CSS variables and trigger network requests, but Manrope overrides `--font-sans` making Geist Sans functionally redundant for body text. All three are loaded regardless.
- Files: `app/layout.tsx`
- Impact: Unnecessary Google Fonts network requests on every page load (3 font families where 1-2 would suffice). Slight performance overhead.
- Fix approach: Decide on one primary font. Remove unused font imports. If Geist is not needed, remove it. If Manrope is not needed, remove it.

**Stale Metadata:**
- Issue: `app/layout.tsx` has `title: "Create Next App"` and `description: "Generated by create next app"` — the default scaffold metadata.
- Files: `app/layout.tsx`
- Impact: Incorrect page title and description in browser tabs, search engine results, and social media previews.
- Fix approach: Update `metadata` in `app/layout.tsx` to reflect the SJA Grading System product name and description.

**Default next.config.ts with No Configuration:**
- Issue: `next.config.ts` is empty — no image domains, no environment variable exposure, no security headers, no redirects, no rewrites.
- Files: `next.config.ts`
- Impact: No production hardening. Missing security headers (CSP, X-Frame-Options, etc.). If the app will use external images or APIs, domains will not be configured.
- Fix approach: Add appropriate `headers()` for security, configure `images.remotePatterns` if external images will be used, and set environment variable exposure as needed.

**`shadcn` Listed as a Runtime Dependency:**
- Issue: The `shadcn` CLI tool (version `^4.1.1`) is listed in `dependencies` in `package.json`, not `devDependencies`. It is a code-generation CLI, not a runtime library. It does not need to be in the production bundle.
- Files: `package.json`
- Impact: Adds CLI tooling to the production `node_modules` install. Wastes install time and disk space in production/CI environments.
- Fix approach: Move `shadcn` to `devDependencies`.

## Security Considerations

**No Security Headers Configured:**
- Risk: Without `next.config.ts` headers configuration, the app ships with no Content-Security-Policy, no X-Frame-Options, no X-Content-Type-Options, etc. Default Next.js adds some headers but CSP must be project-defined.
- Files: `next.config.ts`
- Current mitigation: None configured.
- Recommendations: Add a `headers()` export in `next.config.ts` defining CSP, X-Frame-Options (`SAMEORIGIN`), X-Content-Type-Options (`nosniff`), and Referrer-Policy before production deployment.

**No Authentication Layer:**
- Risk: No auth middleware, no session management, and no protected routes exist. A grading system typically requires role-based access (admin, instructor, student).
- Files: None — not yet implemented.
- Current mitigation: None. Application is not yet deployed with real data.
- Recommendations: Implement authentication before adding any real grading data. Consider Next.js middleware for route protection.

**No Input Validation Layer:**
- Risk: No validation utilities or schema libraries (e.g., Zod, Valibot) are installed. Any future form inputs or API routes will have no validation unless added.
- Files: `package.json` (absence of validation library)
- Current mitigation: None.
- Recommendations: Add a validation library (Zod is common in the Next.js ecosystem) before implementing any API routes or Server Actions.

## Performance Bottlenecks

**Three Font Families Loaded Simultaneously:**
- Problem: `app/layout.tsx` loads Geist Sans, Geist Mono, and Manrope from Google Fonts on every request.
- Files: `app/layout.tsx`
- Cause: Unused holdover from scaffold plus addition of Manrope without removing Geist.
- Improvement path: Reduce to the fonts actually used (likely only Manrope for sans and Geist Mono for mono, or just one family). Each font family removed eliminates a font preload and potentially multiple woff2 file downloads.

- Test coverage: None.

## Dependencies at Risk

**`lucide-react` at Version `^1.7.0`:**
- Risk: `lucide-react` 1.x is a major version ahead of the widely-documented 0.x line. API compatibility with shadcn documentation and community examples may differ.
- Impact: Icon import patterns or icon names may differ from community examples; some icons may not exist or be renamed.
- Migration plan: Verify icon names against the installed version's exports before using them. Lock to a specific minor version once icons are confirmed working.

**`tailwindcss` at Version `^4`:**
- Risk: Tailwind CSS v4 has breaking changes from v3 (no `tailwind.config.js`, CSS-first configuration, new PostCSS plugin). Substantial community documentation, AI training data, and third-party plugin support still targets v3.
- Impact: Third-party component libraries or tutorials written for Tailwind v3 will produce broken output if copied directly. `@tailwindcss/postcss` replaces the old PostCSS plugin. The `tw-animate-css` dependency bridges animation utilities.
- Migration plan: This is intentional — the project started with v4. Always reference Tailwind v4 documentation. Do not use `tailwind.config.js` or v3 plugin syntax.

**`next` at Version `16.2.1`:**
- Risk: Next.js 16 is a major version ahead of widely-documented v14/v15. APIs, file conventions, and behaviors may differ from community resources and AI training data. The AGENTS.md explicitly warns about this.
- Impact: Incorrect code patterns written from v14/v15 knowledge will cause silent or runtime failures.
- Migration plan: Read `node_modules/next/dist/docs/` before writing any Next.js code, as directed by `AGENTS.md`.
