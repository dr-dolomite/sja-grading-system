**New model — SubjectAssignment (prerequisite for ENRL-02):**
One record = "this subject is taught at this grade level (+ optional strand)". `strandId: null` means JHS (applies to all sections of grade level). Non-null means SHS strand-specific.
```
model SubjectAssignment {
  id                  String              @id @default(cuid())
  subjectId           String
  gradeLevelEntryId   String
  strandId            String?
  subject             Subject             @relation(fields: [subjectId], references: [id], onDelete: Cascade)
  gradeLevelEntry     GradeLevelEntry     @relation(fields: [gradeLevelEntryId], references: [id], onDelete: Cascade)
  strand              Strand?             @relation(fields: [strandId], references: [id])
  enrollments         StudentEnrollment[]
  teacherAssignments  TeacherAssignment[]

  @@unique([subjectId, gradeLevelEntryId, strandId])
}
```

**New model — StudentEnrollment (ENRL-02):**
```
model StudentEnrollment {
  id                  String            @id @default(cuid())
  studentId           String
  subjectAssignmentId String
  student             Student           @relation(fields: [studentId], references: [id], onDelete: Cascade)
  subjectAssignment   SubjectAssignment @relation(fields: [subjectAssignmentId], references: [id], onDelete: Cascade)
  createdAt           DateTime          @default(now())

  @@unique([studentId, subjectAssignmentId])
}
```

**New model — TeacherAssignment (ENRL-03):**
Includes `sectionId` to distinguish "Math in G7-Hope" from "Math in G7-Charity" (per D-11 — teacher can teach same subject in multiple sections).
```
model TeacherAssignment {
  id                  String            @id @default(cuid())
  subjectAssignmentId String
  sectionId           String
  teacherId           String
  subjectAssignment   SubjectAssignment @relation(fields: [subjectAssignmentId], references: [id], onDelete: Cascade)
  section             Section           @relation("TeacherAssignments", fields: [sectionId], references: [id], onDelete: Cascade)
  teacher             User              @relation("TeacherAssignments", fields: [teacherId], references: [id])
  createdAt           DateTime          @default(now())
  updatedAt           DateTime          @updatedAt

  @@unique([subjectAssignmentId, sectionId])
}
```

**1. Add two Server Actions to `app/actions/school-structure.ts`:**

`assignSubjectToGradeLevel` — Creates a SubjectAssignment record linking a subject to a grade level entry (+ optional strand for SHS).
- Role check: ADMIN only (extending Phase 2 school structure setup, per D-08)
- Zod schema: `{ subjectId: z.string().min(1), gradeLevelEntryId: z.string().min(1), strandId: z.string().optional() }`
- Prisma: `prisma.subjectAssignment.create({ data: { subjectId, gradeLevelEntryId, strandId: strandId ?? null } })`
- Catch Prisma P2002 (unique constraint) — return `{ errors: { form: ["This subject is already assigned to this grade level and strand."] } }`
- On success: `revalidatePath("/dashboard/school-structure")` and return `{ success: true }`
- State type: `AssignSubjectState = { errors?: { subjectId?: string[], gradeLevelEntryId?: string[], strandId?: string[], form?: string[] }, success?: boolean } | null`

`removeSubjectAssignment` — Deletes a SubjectAssignment record.
- Role check: ADMIN only
- Zod schema: `{ id: z.string().min(1) }`
- Prisma: `prisma.subjectAssignment.delete({ where: { id } })`
- On success: `revalidatePath("/dashboard/school-structure")` and return `{ success: true }`
- State type: `RemoveSubjectAssignmentState = { errors?: { form?: string[] }, success?: boolean } | null`

**2. Create `components/assign-subject-sheet.tsx`:**
