Present the verification checklist to the user and wait for approval. This is a human-verify checkpoint — no code changes are made.

What was built:
- Prisma schema with Student, SubjectAssignment, StudentEnrollment, TeacherAssignment models
- Subject-to-grade-level linking UI on School Structure Subjects tab
- Enrollment page with Students tab (filter, search, create, edit, remove, CSV import)
- Auto-enrollment: creating a student in a section auto-enrolls them in all subjects for that grade level + strand
- Assignments tab with Teacher Assignments table and Adviser Assignments table
- Sheet forms for assigning teachers to subject-section pairs and advisers to sections
- CSV bulk import with template download, validation preview, and batch creation
  </action>
  <verify>
Human completes the verification steps below and types "approved" or describes issues.

**Prerequisites:** Ensure PostgreSQL is running and `.env` has correct DATABASE_URL. Run `npx prisma migrate dev` to apply schema changes. Run `npx prisma db seed` to seed strands. Start dev server with `pnpm dev`.

**ENRL-01 — Student Profiles:**
1. Log in as ADMIN or PRINCIPAL
2. Click "Enrollment & Assignments" in sidebar
3. Click "Enroll Student" button
4. Fill in: LRN (e.g., "100001"), Last Name, First Name, Grade Level (pick one with sections), Section, Sex
5. Verify the auto-enrollment note appears below the Section field
6. Submit — verify toast "Student enrolled successfully" appears
7. Verify student appears in the table with correct data
8. Click Edit (pencil icon) on the student — verify fields are pre-filled
9. Change a field and submit — verify toast "Student updated" and table reflects change
10. Verify grade level and section filter dropdowns work
11. Verify global search finds student by name and LRN

**ENRL-01 — CSV Import:**
12. Click "Import CSV" button
13. Click "Download template" — verify CSV file downloads with correct headers
14. Create a test CSV with 3-5 rows (include one invalid row with missing LRN)
15. Drop or select the CSV file
16. Verify preview table shows valid rows (green badge) and invalid rows (red badge with error)
17. Click "Import N Students" — verify toast shows count
18. Verify imported students appear in the table

**ENRL-02 — Auto-Enrollment (requires School Structure setup first):**
19. Go to School Structure > Subjects tab
20. For a subject, click "Assign to grade levels" — verify Sheet opens
21. Assign a subject to a grade level (check checkbox) — verify it saves
22. Create a new student in a section of that grade level
23. Verify (in database or by implication in Phase 4) that StudentEnrollment records were created

**ENRL-03 — Teacher Assignment:**
24. Go to Assignments tab
25. Verify Teacher Assignments table shows subject-section pairs (requires subjects assigned to grade levels first)
26. Click "Assign Teacher" on a row
27. Select a teacher from the dropdown and submit
28. Verify the teacher name appears in the "Assigned Teacher" column
29. Click "Change Teacher" — verify you can reassign

**ENRL-04 — Adviser Assignment:**
30. Scroll to Adviser Assignments section
31. Verify sections are listed with "Unassigned" badges
32. Click "Assign Adviser" on a section
33. Select an adviser and submit
34. Verify the adviser name replaces the "Unassigned" badge
